📢 Project - 1



👉 국내통합 예약시스템 👈


국내 기차, 버스, 비행기 통합 예약시스템의 부재를
보완 및 추가하여 개발한 국내 통합예약 시스템


1️⃣ 개발 환경

➜ 2S version(platform) : Windows , Mac
➜ JAVA version(Language) : JDK 1.8
➜ Eclipse version(Development tool) : 202103 4.19.0


2️⃣ 개요&목적

2.1 개요
➜ 버스, 기차, 비행기의 통합 교통 노선을 조회 및 예약,결제가 가능하고 다른 기타 서비스를 이용 할 수 있는 프로그램(회원정보조회 , 유실물 조회, 예약내역조회 , 예약취소 , 요금충전)
➜ 한가지의 교통수단이 아닌 통합 교통수단의 노선 조회를 통해 사용자가 원하는 교통수단의 노선의 예약을 수행할 수 있는 프로그램

2.2 목적
➜ 해당 프로그램을 통해 교통수단에 따른 개별 예약 시스템의 불편함을 해소 한다.
➜ 사용자의 입력 정보를 통한 고객 맞춤형 서비스(교통 노선조회/예약/결제, 유실물 조회 등)를 제공한다.


3️⃣ 사용 기술

- 파일 입출력 : 파일 입출력 BufferedReader, Writer(new FileReader, Writer( "path" ))
➜ 컬렉션 : ArrayList, HashMap 등
➜ 오버로딩, 오버라이딩 :생성자 오버로딩, 객체에 속한 ArrayList 함수의 재구현.
➜ 기타 레퍼런스 타입 클래스 : 데이터를 생성하기 위한 Random, Math클래스, 노선 가격 및 일자, 이벤트 표시를 위해 Calendar 클래스를 이용하여 날짜 계산


4️⃣ 핵심 업무

4.1 비회원 : 가입하지 않은 비회원이 이용할 수 있는 기능
➜ 노선 검색/조회 기능, 유실물 조회 기능, 회원가입 기능, 로그인 기능

4.2 회원 : 가입한 회원이 이용할 수 있는 기능
➜ 노선 조회 및 예약 기능, 마일리지 적립 기능, 예약내역 조회/취소 기능, 회원정보 조회/탈퇴 기능, 결제금액 충전 기능


5️⃣ 실행 화면

5.1 메인 화면
➜ 모든 고객이 사용할 수 있는 기능이 있는 메인화면


5.2 노선 조회
➜ 기차, 버스, 비행기중 원하는 교통수단을 선택하고 원하는 노선 선택


5.3 노선 선택
➜ 사용자가 입력한 출발지, 도착지에 따른 교통수단의 노선 목록을 표시


5.4 좌석 선택
➜ 좌석 선택화면 노선 예약 시 해당 교통수단의 좌석 목록을 보여주고 사용자가 원하는 좌석을 입력


5.5 예약 내역 조회
➜ 예약내역의 예약번호를 입력해서 결제 영수증을 출력한다.



📢 Project - 2



👉 공공데이터를 활용한 여행지 앱 👈


공공데이터 파서,파싱을 통해 여행지에 대한 정보를 담고,
카카오맵 API를 통해 도커에 위치를 표시하고, 사용자에게 여행지를 추천해주는 시스템


1️⃣ 개발 환경

➜ 2S version(platform) : Windows , Mac
➜ JAVA version(Language) : JDK 1.8
➜ Android Studio
➜ Adobe Photoshop CS6
➜ Microsoft Excel


2️⃣ 개요&목적

2.1 개요
➜ 흔히 여행을 다니다 보면, 다양한 정보가 필요하고, 정보를 수집하기 위해 다양한 플랫폼들을 이용하고는 한다.
➜ 하지만 그에 대한 정보를 찾기에는 많은 시간이 필요, 다양한 정보가 존재X
➜ 이러한 정보들을 한번에 모두 담은 앱이 있다면, 보편적으로 보다 더 편리함을 제공할 것이라 생각하여 앱을 제작.

2.2 목적
➜ 데이트를 유용하게 할 수 있는 데이트 플랫폼.
➜ 지역에 대한 핫플레이스, 맛집, 카페 등을 추천.
➜ 저역에 대한 주차장, 화장실, 흡연장을 kakao API를 이용하여 도커에 표시 해주어 편의 제공.
➜ 알아본 정보를 이용하여, 일정을 메모하며 스케줄을 짤 수 있는 기능을 제공.


3️⃣ 사용 기술

- XmlPullParser : XML을 파싱 하는 방식으로, XML 구성요소를 순서대로 파싱하면서, 각 이벤트 타입이 발생하고, 타입에 맞는 코드가 실행.
➜ KaKaoMap API : 카카오(KaKao)에서 제공하는 카카오 지도 API, 개발자 등록, 1일 최대 300,000회 사용 가능.
➜ Open API : 서울열린광장에서 데이터의 인증키를 받아, 인증키로 지역별 정보들을 가져와 필요한 데이터들을 파싱.
➜ DBHelper : 사용자가 원하는 정보를 DB에 담을 수 있도록 제공.


4️⃣ 핵심 업무

- 핫플레이스 : 다양한 지역별 핫플레이스들을 담은 공간으로, 지역을 선택하면 해당 지역의 핫플레이스 목록들을 볼 수있다.
➜ 맛집/카페 : 다양한 지역벽 맛집/카페들을 담은 공간으로, 지역을 선택하면 해당 지역의 맛집,카페 목록들을 볼 수 있다.
➜ 주차장 : 카카오맵 API를 통해, 스피너를 통해 원하는 지역을 선택하면 해당 지역의 주차장을 도커를 통해 위치를 나타낸다. (기본 위치 : 현재 자신의 위치)
➜- 화장실 : 카카오맵 API를 통해, 스피너를 통해 원하는 지역을 선택하면 해당 지역의 화장실을 도커를 통해 위치를 나타낸다. (기본 위치 : 현재 자신의 위치)
➜ 흡연장 : 카카오맵 API를 통해, 스피너를 통해 원하는 지역을 선택하면 해당 지역의 흡연장을 도커를 통해 위치를 나타낸다. (기본 위치 : 현재 자신의 위치)
➜ 일정메모 : DBHelper를 통해, 사용자가 달력에 원하는 정보를 입력하면 그 정보를 DB에 저장하여 원활히 스케줄을 짤 수 있다.


5️⃣ 실행 화면

5.1 로딩 화면
➜ 처음 시작하였을 때 구동되는 로딩 화면



5.2 메인 화면
➜ 모든 사용자가 사용할 수 있는 기능이 있는 메인화면



5.3 핫플레이스/맛집
➜ 사용자에게 핫플레이스/맛집들을 추천해주는 기능



5.4 흡연장/주차장/화장실
➜ 사용자에게 흡연장/주차장/화장실의 위치를 알려주는 기능



5.5 일정 메모
➜ 사용자가 원하는 날짜에 원하는 일정을 메모할 수 있는 기능

👉 나만의 포트폴리오 👈


현재 보여지고 있는 해당 사이트로, 나의 정보들을 담을 수 있는 나만의 포트폴리오 공간


1️⃣ 개발 환경

➜ 2S version(platform) : Windows , Mac
➜ Visual Studio Code ➜ Eclipse, Chrome ➜ Google Apps Mail


2️⃣ 개요&목적

2.1 개요
➜ 현재 보여지고 있는 사이트로, 나만의 정보들을 담을 수 있는 공간이며, 다른 사람들에게 자신의 커리어를 보여주며, 관심 있는 기업이나, 사람들이 연락을 할 수 있도록 한다.

2.2 목적
➜ 웹사이트에 자신의 정보를 담고, 다른 이들에게 웹사이트를 공유하여 관심이 있는 사람들에게 연락을 받도록 서비스를 제공한다.


3️⃣ 사용 기술

➜ HTML5 : HTML 4.01, XHTML 1.0, DOM 레벨 2 HTML에 대한 차기 표준 제안.
➜ CSS : 문서의 스타일를 꾸밀 때 사용하는 스타일 시트 언어.
➜ JavaScript : 멀티-패러다임 언어로 명령형, 함수형, 객체 지향형 언어.
➜ jQuery : 자바스크립트를 쉽게 활용할 수 있도록 도와주는 오픈소스 기반의 자바스크립트 라이브러리.
➜ BootStrap : 전세계적으로 가장 많이 사용되는 오픈소스 CSS 라이브러리.


4️⃣ 핵심 업무

➜ main-menu : 메인 화면의 구성 및 메뉴 구성.
➜ About me : 자신에 대해서 설명하고, 의지를 나타내는 공간.

➜ 사람들이 자신의 능력을 최대한 활용하고, 더 많은 기회를 만나도록 돕는 것이 궁극적인 목표이다.

2.2 목적
➜ 사람들이 자신의 재능과 능력을 광범위한 일자리 세계에서 활용하고, 새로운 도전을 이어나가도록 돕는 시스템 개발.

4️⃣ 핵심 업무

➜ 메인, 로그인, 회원가입 : 메인 배너 구현 및 로그인 유효성 검사, 회원 가입 유효성 검사, Security로 해당 유저의 권한을 부여한다.
➜ 인재 둘러보기 : 구직 신청이 완료된 회원은 구직 시장에 등록된다. 사업자는 해당 인재들중 원하는 인재를 선택하여 선택 및 결제를 하여 구인을 할 수 있다. 결제는 카카오페이로 진행된다.
➜ 들어볼텨 : 구직을 하였던 회원들이, 후기를 작성할 수 있는 CRUD 게시판.
➜ 말해볼텨[사업자 or 구직회원] : 사업자, 구직회원으로 게시판이 분류되어 있고, 자유롭게 대화를 할 수 있는 CRUD 게시판.
➜ 포트폴리오 - 자신이 진행하였던, 프로젝트나 일하였던 사항들을 기록할 수 있는 게시판, 사업자는 이를 통해 사용자의 포트폴리오를 열람할 수 있다.
➜ 구직회원 마이페이지 - 프로필, 문의내역, 포트폴리오, 지원내역, 들어볼텨, 말해볼텨 등 CRUD가 가능한 마이페이지.
➜ 사업자 마이페이지 - 사업자 프로필, 사업자 상세 프로필, 위시 리스트, 리뷰, 나의 문의내역, 주문 내역, 말해볼텨 등 CRUD가 가능한 마이페이지.

6.2 로그인
➜ 로그인은 스프링 시큐리티 기능을 사용하였습니다.
➜ 유저 정보 객체는 customUserDetailsService를 직접 구현하여 구인회원, 구직회원, 관리자 이렇게 3가지를 구분합니다.
➜ 각각 확인하는 쿼리문을 날려 어떤 유저인지 확인하고 그에 맞는 계정 정보(DTO)를 반환해주고 권한(Role)을 부여해줍니다.
➜ 로그인 실패시 실패 메시지를 보여주고
➜ 비밀번호는 BCrypt로 암호화하여 처리합니다.
➜ 시큐리티의 로그인 기능으로 인해
➜ 컨트롤러의 메소드들에 @PreAuthorize 어노테이션을 붙여 로그인 및 권한으로 접근제한을 해줍니다.



6.3 회원가입
➜ 회원가입은 각 폼컨트롤의 값에 대해 유효성 검사를 합니다.
➜ 아이디 검사는 유효성 검사 외에 중복검사를 ajax로 처리합니다.
➜ 아이디는 구인회원, 구직회원, 관리자를 모두 검사 하여 세 테이블 중에 아이디가 하나라도 존재하면 중복으로 체크합니다.
➜ 모든 값이 유효할 때에 회원가입 버튼을 누르면 가입이 완료됩니다.



6.4 인재 둘러보기
➜ 관리자로부터 승인된 구직 신청서만 리스트에 출력됩니다.
➜ 필터를 통해 원하는 분야의 구직신청서만 볼 수 있습니다.
➜ 상세 페이지에서는 인재의 급여, 학력, 경력, 자기소개서를 볼 수 있습니다.
➜ 찜을 하면 위시리스트에서 볼 수 있으며 현재 인재와 분야가 같은 다른 구직자들의 정보도 볼 수 있습니다.



6.5 결제
➜ 결제는 아임포트 API를 활용하여 구현했습니다.
➜ 카카오페이를 통해 결제가 진행되고 결제가 완료되면 주문정보를 DB에 저장합니다.
➜ 그 다음 이름, 직무, 기간 결제금액에 정보를 사용자에게 출력하여 확인시킵니다.



6.6 들어볼텨
➜ 구직 후기 게시판인 들어볼텨는 구직을 진행하였던 회원에 한에 글을 작성할 수 있는 공간입니다.
➜ 기본적인 CRUD 기능과 페이징 기능 등이 있습니다.



6.7 말해볼텨
➜ 말해볼텨는 사업자 회원과 구직회원의 게시판으로 분리되어 있으며, 사업자와 구직회원은 서로의 글을 볼 수 없습니다.
➜ 기본적인 CRUD 기능과 페이징 기능 등이 있습니다.



6.8 포트폴리오
➜ 포트폴리오 둘러보기 페이지는 모든 구직자들의 포트폴리오를 볼 수 있는 페이지이며 각 글을 클릭하면 포트폴리오 내용, 이미지 등 상세 정보가 나오게 됩니다.
➜ 왼쪽 상단의 select 박스를 통해 직군 별로 필터링 된 포트폴리오 정보를 볼 수 있습니다.
➜ DB에서 페이징 처리된 포트폴리오 정보를 가져오기 위한 쿼리문과 직군 별로 포트폴리오를 조회하기 위한 쿼리문 등이 정리되어 있습니다.



6.9 구직회원 마이페이지
➜ 구직회원 전용 마이페이지로 마이페이지에 관한 기능들을 담았습니다.
➜ 문의내역, 포트폴리오, 지원내역, 회원정보 수정등이 있습니다.



6.10 사업자 마이페이지
➜ 사업자 회원 전용 마이페이지로 마이페이지에 관한 기능들을 담았습니다.
➜ 사업자 프로필, 위시리스트, 나의 문의내역, 주문 내역, 커뮤니티 등이 있습니다.
